# SNL 编译器设计思路

本文档记录当前项目重命名后的文件职责，以及编译器从源程序到语义检查的完整流程。

## 一、文件命名规则

项目文件已经按照编译器流程统一加上序号前缀：

- `1.` 表示最先执行的词法分析阶段。
- `2.` 表示语法分析阶段。
- `3.` 表示语义分析阶段。
- 后续文件是测试、界面和 LL(1) 辅助工具。

由于 Python 不能用普通 `import` 直接导入 `1.lexer.py` 这种带数字和点号的模块名，所以相关文件内部使用 `importlib` 按文件路径加载依赖模块。这样既保留了清晰的顺序命名，也能保证程序正常运行。

## 二、当前项目结构

| 文件 | 说明 |
| --- | --- |
| `1.lexer.py` | 词法分析器。定义 `TokenType`、`Token`、`Tokenizer`，负责把 SNL 源代码转换成 Token 序列。 |
| `2.recursive_descent_parser.py` | 递归下降语法分析器，也是当前主流程使用的语法分析器。文件前半部分定义 AST 节点，后半部分实现语法分析函数。 |
| `3.semantic_analyzer.py` | 语义分析器。负责建立符号表、管理作用域、检查重复定义、未声明标识符、类型不匹配、过程调用参数等语义错误。 |
| `4.semantic_tests.py` | 语义测试入口。构造 SNL 源程序，依次执行词法分析、语法分析、语义分析，并断言期望错误。 |
| `5.snl_frontend_app.py` | Tkinter 图形界面。提供源码输入框、Token 表、语法树显示等功能。当前主要连接词法分析和递归下降语法分析。 |
| `6.predict_set_builder.py` | Predict 集生成脚本。根据内置文法计算 Predict 集，并输出 `predict_table.md`。 |
| `7.ll1_table_builder.py` | LL(1) 分析表生成脚本。计算 FIRST/FOLLOW/Predict，并生成 `ll1_table.json`。 |
| `8.ll1_parser.py` | LL(1) 表驱动语法分析器。读取 `ll1_table.json`，根据 Token 序列构造 AST。 |
| `9.grammar_debug.py` | 文法调试辅助脚本。目前主要用于临时检查文法符号。 |
| `10.recursive_descent_parser_backup.py` | 递归下降语法分析器历史备份。当前主流程不依赖它。 |
| `ll1_table.json` | LL(1) 分析表数据文件，由 `7.ll1_table_builder.py` 生成，被 `8.ll1_parser.py` 读取。 |
| `predict_table.md` | Predict 集表格文件，由 `6.predict_set_builder.py` 生成。 |
| `SNL文法/` | SNL 文法参考图片目录。 |

## 三、编译器主流程

当前项目的主要编译流程是：

```text
SNL 源代码
  |
  v
1.lexer.py
  Tokenizer.tokenize()
  生成 Token 序列
  |
  v
2.recursive_descent_parser.py
  syntax_analysis(tokens)
  生成 AST 语法树
  |
  v
3.semantic_analyzer.py
  semantic_analysis(program)
  建立符号表并检查语义错误
  |
  v
输出符号表或错误列表
```

## 四、测试流程

语义测试入口是：

```text
4.semantic_tests.py
  |
  v
load_stage_module("make_token", "1.lexer.py")
  |
  v
load_stage_module("syntax", "2.recursive_descent_parser.py")
  |
  v
load_stage_module("semantic_analysis", "3.semantic_analyzer.py")
  |
  v
执行 SemanticCase 测试用例
```

运行方式：

```bash
python .\4.semantic_tests.py
```

当前已有测试：

- 重复变量定义：`integer value; char value;`

## 五、GUI 流程

图形界面入口是：

```text
5.snl_frontend_app.py
  |
  v
用户输入 SNL 源代码
  |
  v
1.lexer.py
  生成 Token 表
  |
  v
2.recursive_descent_parser.py
  生成 AST 语法树
  |
  v
界面显示 Token 表和语法树
```

运行方式：

```bash
python .\5.snl_frontend_app.py
```

注意：GUI 里的“语义分析”和“中间代码生成”标签页目前仍是占位内容，还没有真正接入 `3.semantic_analyzer.py`。

## 六、LL(1) 辅助流程

LL(1) 相关文件是辅助路线，不是当前主测试入口。

```text
6.predict_set_builder.py
  计算 Predict 集
  输出 predict_table.md

7.ll1_table_builder.py
  计算 FIRST/FOLLOW
  生成 ll1_table.json

8.ll1_parser.py
  读取 ll1_table.json
  表驱动分析 Token 序列
  构造 AST
```

## 七、语义分析支持的检查

`3.semantic_analyzer.py` 当前支持：

- 同一作用域内类型、变量、过程不能重复定义。
- 过程参数不能重复定义。
- 使用变量前必须声明。
- 使用类型前必须定义。
- 调用过程前必须定义。
- 数组下标必须是整数。
- 记录字段必须存在。
- 赋值语句左右类型必须一致。
- 算术表达式操作数必须是整数。
- 关系表达式左右类型必须兼容。
- 过程调用参数个数和类型必须匹配。
- `var` 参数必须传入变量，不能传入常量或表达式。
